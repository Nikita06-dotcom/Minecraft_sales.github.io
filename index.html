Справка программы Продажи Minecraft 2.0.0

\|/

ЧТО МОЖЕТ ПРОГРАММА

Программа позволяет упростить расчёты для выставления на продажу внутри Minecraft сервера. Например, сервер Holyworld имеет такую механику.

Сложность заключается в том, что перед выставлением товара на ah (магазин внутри сервера) нужно просчитать все до мелочей. В этих проблемах помогает данная программа.

 

РАЗДЕЛЫ ПРОГРАММЫ

Программа делится на два варианта счета: ПРОСТОЙ и НАСТРАИВАЕМЫЙ.

ПРОСТОЙ

Для этого выбора счёта используйте первый вариант - «Простой» в разделе «Выберите вариант счета». ВАЖНО!!! НЕЛЬЗЯ, ЧТОБЫ ДВА ФЛАЖКА ИЗ РАЗДЕЛА «ВЫБЕРИТЕ ВАРИАНТ СЧЕТА» БЫЛО АКТИВНО, НУЖНО ИСПОЛЬЗОВАТЬ ТОЛЬКО ОДИН ВАРИАНТ.

После нужно выбрать, как будут считаться предметы для продажи. Т. е. можно выбрать счет по стакам (в 1 стаке 64 предмета) или суммарно кол-во всех предметов для продажи. За это отвечают флажки «Стаками» и «Сумма предметов» соответственно.

После этого нужно вписать ЦЕЛОЕ значение в поле «Кол-во предметов» с учётом той категории, которую вы выбрали.

Ниже есть другое поле ввода: «Цена за 1 единицу товара». В него нужно вписать цену за 1 единицу товара. Например, 22,56 (Вещественные числа писать только через запятую).

Сейчас приведу пример работы программы. Нужно выставить на продажу 44 предмета по 30,8 монет за шт. Для этого мы поставим флажок в варианте «Простой» и только там. Затем выберем флажок «Сумма предметов» и впишем 44 в поле «Кол-во предметов», затем 30,8 в «Цена за 1 единицу товара». И Нажмём кнопку «Рассчитать». Программа выдаст результат в окне «Memo 1». Там есть вся нужная информация и команды для копирования в саму игру.

*В другие поля автоматически подставляются значения. Их не нужно менять.

Аналогично это работает и со стаками. Например, нужно выставить 2 стака вещей по 23 монеты за штуку на ah. Выполняем все подготовительные процедуры (активные флажки: «Простой», «Стаками». Остальные неактивны) и вписываем 2 в поле «Кол-во предметов», а 23 в «Цена за 1 единицу товара». И Нажимаем кнопку «Рассчитать». Программа выдаст результат в окне «Memo 1». Там есть вся нужная информация и команды для копирования в саму игру.

___

*В данном случае флажок «Приписка full к командам» активен => команды поменялись.

НАСТРАИВАЕМЫЙ

Для этого выбора счёта используйте второй вариант - «Настраиваемый» в разделе «Выберите вариант счета». ВАЖНО!!! НЕЛЬЗЯ, ЧТОБЫ ДВА ФЛАЖКА ИЗ РАЗДЕЛА «ВЫБЕРИТЕ ВАРИАНТ СЧЕТА» БЫЛО АКТИВНО, НУЖНО ИСПОЛЬЗОВАТЬ ТОЛЬКО ОДИН ВАРИАНТ.

После нужно выбрать, как будут считаться предметы для продажи. Т. е. нужно выбрать сколько в 1 стаке предметов. Можно самому ввести ЦЕЛОЕ значение в поле «Кол-во в 1 стака пр-ов» или нажать на кнопку «16», для принятия одного стака за 16 предметов. Аналогично работает кнопка «64».

Ниже этого поля ввода есть другое: «Кол-во стаков». Тут не буду объяснять. Правее – «Ц. за X пр-ов». В него пишется цена за X предметов, где X берётся из самого правого поля ввода «X». Еще ниже есть поле «Кол-во ост…». В него вписывается кол-во оставшихся предмето.

Сейчас приведу пример использования программы. Например, нужно выставить на продажу 44 стака, где один стак равен 16 и 23 шт. по 30,8 монет за 4 шт.

*Сообщение "до 64" можно игнорировать. Также, если флажок «Приписка full к командам» активен, то появятся приписки в конце команд.
